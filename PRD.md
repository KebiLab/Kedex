# Product Requirements Document (PRD) — Kedex
**Автор:** KebiLab  
**Продукт:** Kedex (Аналог OpenAI Codex Desktop App с поддержкой любых LLM-провайдеров и голосовым вводом)  
**Статус:** Draft v1.1  

---

## 1. Обзор продукта
**Kedex** — это десктопное приложение для ОС macOS и Windows, предоставляющее среду для автономной разработки с помощью ИИ-агентов. Приложение позволяет разработчикам делегировать сложные, многошаговые инженерные задачи ИИ-агенту, сохраняя полный контроль над процессом через систему планов, интерактивного утверждения действий и визуального контроля изменений.

### Ключевое отличие от оригинального Codex:
Полная независимость от провайдера ИИ. Kedex поддерживает работу с любыми популярными коммерческими API (OpenAI, Anthropic, Google Gemini, Mistral, DeepSeek), локальными моделями (через Ollama/Llama.cpp) и любыми кастомными OpenAI-совместимыми эндпоинтами.

---

## 2. Архитектура системы (Стек)
Приложение строится на базе трехслойной архитектуры:
1. **Renderer Layer (Frontend):** React 18, Tailwind CSS, Radix UI (компоненты), Framer Motion, Shiki (подсветка синтаксиса), ProseMirror (редактор).
2. **Main Process (Desktop Bridge):** Electron, Node.js. Отвечает за сопряжение интерфейса с ОС, управление окнами, работу с базой данных SQLite (`better-sqlite3`) и запуск сессий терминала (`node-pty`).
3. **Core Layer (Agent Engine):** Скомпилированный бинарный файл на **Rust** (`kedex-core`). Выполняет тяжелые задачи: парсинг кодовой базы (`tree-sitter`), сетевые запросы через `reqwest`/`hyper` (для быстрой потоковой передачи токенов), обработку локального контекста и поддержку протокола MCP (Model Context Protocol).

*Общение между Electron (Node) и Rust Core происходит по протоколу JSON-RPC через стандартные потоки ввода-вывода (stdio).*

## 3. Функциональные требования

### 3.1. Управление провайдерами и гибкая настройка моделей (Provider & Model Manager)
* **Предустановленные провайдеры:** OpenAI, Anthropic, Google Gemini, DeepSeek.
* **Локальный провайдер:** Ollama (автоматическое обнаружение локальных инстансов).
* **Кастомный провайдер (Custom Endpoint):** Возможность ввести любой `Base URL`, `API Key` и имя модели (совместимо с OpenAI API).
* **Настройки для каждой модели:** Температура, лимиты контекстного окна (Context Window Limit), максимальное количество токенов на вывод.

### 3.2. Многопоточная рабочая область и Git/GitHub интеграция
* **Проекты (Workspaces):** Пользователь открывает локальную папку с проектом.
* **Сессии (Threads):** Внутри проекта можно создавать независимые чаты (сессии) под разные задачи.
* **Параллельные ветки (Worktrees):** Kedex-core умеет создавать изолированные копии рабочего каталога (виртуальные или реальные git-worktrees), чтобы агент мог писать код и тестировать его, не мешая основной работе пользователя в IDE.
* **Интерактивный Diff:** Визуализация изменений кода в стиле Git Side-by-Side прямо в приложении перед их фиксацией.
* **Авто-коммиты и синхронизация:** Kedex имеет встроенную интеграцию с локальным Git и утилитой GitHub CLI (`gh`). Он умеет автоматически создавать коммиты с описанием проделанной работы по стандарту Conventional Commits и отправлять изменения в удаленный репозиторий (push) после успешного завершения каждого этапа плана.

### 3.3. Двухрежимное выполнение задач (Plan & Goal Modes)
* **Plan Mode (Режим планирования):**
  * Агент анализирует задачу и создает пошаговый план (`.kedex/plans/task_id.json`).
  * Функция **Grill-Me:** Агент задает пользователю уточняющие вопросы, если в задаче есть неопределенность, прежде чем начать писать код.
  * Пользователь может вручную редактировать шаги плана, менять их местами или удалять.
* **Goal Mode (Режим выполнения):**
  * Агент последовательно выполняет утвержденный план.
  * Самопроверка: агент запускает сборку/тесты в песочнице, перехватывает ошибки компиляции и сам их исправляет (цикл саморефлексии).

### 3.4. Браузер и UI-аннотации (UI Annotation Workflow)
* Встроенный headless-браузер на базе Chromium (через Chrome DevTools Protocol в Electron).
* Возможность сделать скриншот запущенного веб-приложения непосредственно в Kedex.
* **Инструмент разметки:** Пользователь может рисовать поверх скриншота, выделять области багов и писать комментарии.
* Агент считывает скриншот, координаты разметки, DOM-дерево страницы и вносит соответствующие правки в CSS/HTML/JS.

### 3.5. Голосовой ввод (Voice Input)
* Интеграция кнопки записи голоса в поле ввода.
* **Опции транскрипции:** Использование облачного Whisper API (с возможностью ввода кастомного ключа) или локальной модели транскрипции (Whisper.cpp / Web Audio API).
* Голосовые команды переводятся в текст и мгновенно отправляются агенту.

### 3.6. Безопасность и подтверждения (Approval Flows)
* По умолчанию все потенциально опасные действия (выполнение терминальных команд типа `npm install` или `rm`, изменение файлов вне рабочей директории) требуют явного подтверждения пользователя («Разрешить» / «Запретить»).
* Возможность настроить авто-разрешение для безопасных команд чтения файлов.

## 4. Нефункциональные требования
* **Низкая задержка:** Использование Rust для парсинга файлов гарантирует, что индексация репозитория размером более 10 000 файлов занимает не более 5 секунд.
* **Конфиденциальность:** Локальные ключи шифруются и хранятся в системном хранилище (Keychain для macOS, Credential Manager для Windows) через интеграцию Rust-модуля `keyring`.
* **Автономность:** Возможность работы полностью локально при выборе Ollama и локального Whisper.
