Внимательно изучи требования и начни пошаговую генерацию проекта. Не пиши всё в одном файле. Разделяй код на логические модули.

### ЭТАП 2: Создание моста обмена сообщениями (Electron <-> Rust)
Реализуй в Rust-модуле `core/src/main.rs` чтение из `stdin` и запись в `stdout` по протоколу JSON-RPC. 
Реализуй в `src/main/rust-bridge.ts` запуск бинарника Rust как дочернего процесса (child_process.spawn) и двунаправленный обмен сообщениями. Напиши типизацию для сообщений (запросы на чтение файлов, создание плана, выполнение команды терминала).
*После успешной настройки моста сделай коммит "feat: implement JSON-RPC bridge between electron and rust" и запушь его.*

### ЭТАП 3: Модуль провайдеров и настроек (Provider Manager)
Напиши React-компонент `SettingsModal.tsx` и соответствующий стейт-менеджер. В настройках должна быть вкладка "Providers", где пользователь может:
1. Выбрать провайдера из списка (OpenAI, Anthropic, Gemini, DeepSeek, Ollama, Custom).
2. Ввести API Key.
3. Для Custom ввести Base URL и произвольное имя модели.
Реализуй сохранение этих настроек в локальную БД через Electron `better-sqlite3`. Напиши Rust-клиент в `core/src/agent.rs`, который считывает эти настройки и отправляет потоковые (streaming) запросы к выбранному провайдеру, используя стандартный OpenAI-совместимый API или Anthropic API.
*После реализации настроек сделай коммит "feat: add multi-provider settings and sqlite persistence" и запушь его.*

### ЭТАП 4: Голосовой ввод (Voice Input)
Реализуй компонент `VoiceInput.tsx` в виде иконки микрофона рядом с полем ввода. При нажатии используется Web Audio API для записи голоса в буфер. 
Настрой отправку аудиофайла на бэкенд Electron, где он будет транскрибироваться:
- Либо через вызов OpenAI Whisper API (если у пользователя есть ключ).
- Либо через отправку на локальный сервер Ollama / Whisper.cpp.
Полученный текст должен автоматически подставляться в поле ввода чата.
*После реализации голосового ввода сделай коммит "feat: implement voice input with Whisper API and local fallback" и запушь его.*

### ЭТАП 5: Режимы Plan Mode и Goal Mode
1. В `core/src/agent.rs` опиши структуру Плана: шаги, статус (pending, running, success, failed), зависимости.
2. Реализуй генерацию плана моделью по текстовому запросу. Агент должен сначала создать файл `.kedex/plans/plan_[timestamp].json`.
3. Напиши UI-компонент `PlanViewer.tsx`, который выводит шаги плана в виде интерактивного списка чекбоксов с возможностью редактирования текста шага и изменения приоритета.
4. Реализуй переход в `Goal Mode`: когда план утвержден, агент начинает итеративно запускать шаги. Если шаг требует выполнения команды, Electron отправляет её в PTY (`node-pty`) и возвращает вывод в Rust Core. Если шаг завершился с ошибкой, агент должен автоматически переписать код и запустить шаг заново.
*После завершения работы над режимами планирования и выполнения сделай коммит "feat: implement plan and goal execution modes with PTY terminal integration" и запушь его.*
